= Restore workloads with clean restore in NetApp Ransomware Resilience
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ./media/

[.lead]

With clean restores, NetApp Ransomware Resilience enables you to clean ransomware-affected workloads before restoring them. Wit clean restores, you create an isolated recovery environment that sequesters the workload to identify affected files and optimized recovery methods to minimize data loss. 

The clean restore option is an alternative to the xref:rp-use-recover.html[custom restore option]. 

== Considerations

// * Number of isolated recovery environments per console agent
* Clean restore is supported on AWS in the US East 1 region.
* If the isolated recovery environment doesn't have capacity for a new operation, it's queued until there is availability. 
* When you configure an isolated recovery environment, it's connected to a Console agent. Workloads must be cleaned in an isolated recovery environment hosted on the same Console agent. 
* You can monitor the status of active and queued clean restore operations at any time in the **Recovery** dashboard of Ransomware Resilience. 

== Create an isolated recovery environment

. In Ransomware Resilience, select **Settings**. 
. In the Clean restores card, select **Add** if this is your first environment or **Manage** if you've already created one. 
. If this is your first time creating an isolated recovery environment, proceed to the next step. 
If you've already created an isolated recovery environment, select **Add** to create another one. 
. Provide the follow inputs: 
    * A **Name** for the environment. 
    * Permissions 
    * Provide the authentication credentials for your clean restore environment, including **Access key** and **Secret key**. 
+
image:screenshot-set-up-recovery-environment.png[Screenshot of isolated recovery environment authentication set-up.]
+
. Select **Add** to create the isolated recovery environment.
+
You can track the progress in the **Settings** tab. Once the room is created, you can select **Manage** to view the clean room.

== Perform a clean restore

. In the **Recovery** section, select the workload you want to restore. Select **Restore**. 
. In the Restore Type options, select **Clean restore** then **Next**. 
. If you haven't created an isolated recovery environment for this system, provide the necessary information to link:#create-an-isolated-recovery-environment[create the isolated recovery environment]. Select **Next**. 
. If you have created an isolated recovery environment, select the isolated recovery environment you want to use.  
. Select **Restore**. 
// The isolated recovery environment is automatically selected if one is already connected to the Console agent. Otherwise, you can select an isolated recovery environment
. If you haven't peered the isolated recovery environment, accept the peering request in the production environment.
. After the peering request is accepted, Ransomware Resilience perform a set up. Once the set up completes successfully, Ransomware Resilience denotes **Setup complete**. To proceed to data analsysis, select **Next**. 
. After the setup completes, select **Run analysis**. With the analysis, Ransomware Resilience investigates all restore points—snapshots and backups—from the seven days prior to the incident. Ransomware Resilience analyzes the restore points, noting which ones are potentially impacted.
+
Once a restore point is discovered without impacted files, all earlier restore points are skipped. For example, if the ransomware event occured on October 10, and a restore point was discovered on October 7 at 10:21 AM with no impacted files, all restore points prior to October 7 at 10:21 AM are skipped. 
+
image:screenshot-clean-room-analysis.png[Screenshot of the clean restore analysis step.]

. After the analysis completes, select **Next** to plan your recovery. Ransomware Resilience presents two options: **Least data loss** and **Latest unimpacted restore point**. Select either options, and optionally look at individual file data for either restore point. 
+
To view granular file event data, select a file to analyze its encryption status; when it was created, modified, or deleted, and what restore point corresponds to what action. 
. After you choose your restore point, select **Create restore point** to begin cleaning your files. 
. Ransomware Resilience begins cleaning the workload. 
//If cleaning of any file fails, those files are excluded from the recovery. Select if you want to quarantine those files by saving them to another location. 
+
Once the cleaning completes, select **Next** to initiate the recovery. 
. Choose where to save the original workload: the original location or an alternate. 
+
If you choose the original location, select whether you want to save a new copy or replace the existing.
+
If you select an alternative location, select the **System** and **Storage VM** then enter the **Path for the new volume** and the **New volume name**. 
. Select **Start recovery** to initiate the recovery. 
. Once the recovery completes, you can release the resource based on the save location from the previous step. Select **Release resources and end** to initiate this process. To confirm you want to release the resource, select **End**. 

== Delete a clean room

You can't delete a clean room with an restoration in progress. You can cancel the restore operation or wait until the restore completes then delete the clean room. 

. Go to **Settings**.
. In the Clean restore card, select **Manage**. 
. Identify the clean room you want to delete. In the actions menu column for that room, select the three dots `...`.
. Select **Delete**.
. Select **Delete** to confirm the operation.
