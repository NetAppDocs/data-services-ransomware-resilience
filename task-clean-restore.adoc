= Restore workloads with clean restore in NetApp Ransomware Resilience
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ./media/

[.lead]
With a clean restore, NetApp Ransomware Resilience enables you to create an isolated recovery environment that sequesters the workload to identify and clean affected files. The clean restore identifies optimized recovery paths to minimize data loss and get your workloads back online in the shortest amount of time possible.

The clean restore option is an alternative to the xref:task-custom-restore.html[custom restore option]. 

[NOTE]
Documentation for clean recovery is provided as a technology preview. With this preview offering, NetApp reserves the right to modify offering details, contents, and timeline before General Availability. 

== Overview 

Clean restore guides you through multi-step process that optimizes recovery. 

* **SETUP**: To perform a clean restore, you must first create an isolated recovery environment then select the workload to restore. Ransomware Resilience creates a clean room where all actions for the workload are taken. 
* **ANALYSIS**: In the clean room, Ransomware Resilience analyzes all snapshots to see if encryption is present. Clean restore determines a recovery map to determine the optimal recovery plan. 
+
Once a restore point is discovered without impacted files, all earlier restore points are skipped. For example, if the ransomware event occurred on October 10, and a restore point was discovered on October 7 at 10:21 AM with no impacted files, all restore points prior to October 7 at 10:21 AM are skipped. 
+
The analysis step also displays the number of files reviewed, denoting which are impacted and which are not. 
* **PLAN**: Choose the provided options for recovery:
** _Latest unimpacted restore point_: the quickest restore point from the most recent unecrypted snapshot from before the attack
** _Least data loss_: the most recent version of of every unecrypted file from different snapshots
+
You can also view file histories in the planning step, to see when ransomware events impacted the files and how the recovery point relates in time to that event. 
* **CLEANING**: Ransomware Resilience removes malware from the recovery point. If any files can't be cleaned, they are excluded from the recovery and quarantined to a separate location. 
* **RESTORE**: Ransomware Resilience restores the clean data into the source environment. 
* **END**: Ransomware Resilience provides a detailed summary of the process and closes the clean room.

== Considerations

* Clean restore is currently only supported for NAS file shares (NFS/CIFS).
* Clean restores must be restored to the source environment. 
* You can only perform a clean restore for an encryption-based ransomware attack. 
// * Number of isolated recovery environments per console agent
* If the isolated recovery environment doesn't have capacity for a new operation, it's queued until there is availability. 
* You can monitor the status of active and queued clean restore operations at any time in the Ransomware Resilience Recovery dashboard. 
* When you initiate a clean resotre, the original volume is unmounted, which can disrupt IO access. 
* Enabling clean recovery may incur third party cloud provider charges, some of which may reoccur until the environments you create with this feature are deleted. Examples of third-party charges include the creation and deployment of compute instances and additional production storage capacity for cleaning and recovery. For example:
+
For workloads that needs to be recovered, twos AWS EC2 instances are deployed in the clean room within the isolated recovery environment for cleaning encryption and malware.
+
Clean recovery in Cloud Volumes ONTAP instances also creates metadata volumes and snapshot clones for analysis.

== Create an isolated recovery environment

Before conducting a clean restore, you must create an isolated recovery room. 

The process for creating an isolated recovery room differs if the environment is in the cloud or on-premises. Ensure you follow the instructions for the correct location. 

[role="tabbed-block"]
====
.Cloud
--
. In Ransomware Resilience, select **Settings**. 
. In the Clean restores card, select **Add** if this is your first environment or **Manage** if you've already created one. 
. If this is your first time creating an isolated recovery environment, proceed to the next step. 
If you've already created an isolated recovery environment, select **Add** to create another one. 
. For a cloud-based isolated recovery environment, select **Amazon Web Services**. 
. Expand the **Prerequisites** section then select **View prerequisites**. Choose the architecture of your isolated recovery environment (for example, cloud to cloud), then review the prerequisites. Ensure you've satisified all of them. 
. Expand the **Name** section. Create a **Name** for the environment. 
. Expand the *Systems* section. Choose the systems that you want to connect to the isolated recovery environment. For each system you select, you must provide the storage VM's IP address, subnet mask, and gateway.
+ 
image:clean-restore-systems.png[Screenshot of systems selection interface.]
. Expand the **Authentication** section. 
    * Enter the cloud **Account** ID and select the **Region** for the account. 
    * Enter the **Access key** and **Secret key** for the account**. 
* Expand the *Virtual private cloud (VPC)* section. Enter the VPC ID where the isolated recovery environment is to be deployed. 
+
image:screenshot-set-up-recovery-environment.png[Screenshot of isolated recovery environment authentication set-up.]
+
. Select **Add** to create the isolated recovery environment. Ransomware Resilience tests the connection after you select add; This process can take a while.

--
.On-premises
--
. In Ransomware Resilience, select **Settings**. 
. In the Clean restores card, select **Add** if this is your first environment or **Manage** if you've already created one. 
. If this is your first time creating an isolated recovery environment, proceed to the next step. 
If you've already created an isolated recovery environment, select **Add** to create another one. 
. In the location block, select **On-premises**. 
. Expand the **Prerequisites** section then select **View prerequisites** then review the prerequisites. Ensure you've satisified all of them. 
. Expand the *Name* section. Assign a **Name** for the isolated recovery environment. 
. Expand the *Systems* section. Choose the systems that you want to connect to the isolated recovery environment. Assign a storage VM IP, subnet mask, and gateway for each system that has workloads to be recovered in the isolated recovery environment.  
+
image:clean-restore-systems.png[Screenshot of systems selection interface.]
. Expand the **Console agent** section. Select the Console agent from the dropdown menu that's deployed in the isolated recovery environment. 
. Expand the *Compute* section. 
.. Provide the credentials to authenticate the vCenter server: *IP address* or full-qualified domain name (FQDN), *Port*, *User name*, and *Password*.
.. Select **Authenticate** to confirm the credentials. You must receive confirmation hat the credentials are authenticated before proceeding. 
.. Select the *Datacenter*, *Cluster*, and *ESXi host* where the VMs will be deployed.
.. Select the ESXi hosts *Datastore* and *Network*.
.. Enter the *IP address range* and *Gateway* for the VMs. 
+
image:clean-restore-compute.png[Screenshot of compute selection interface.]
. Expand the *VMs* section, which is populated based on your inputs in the *Compute* section. Select the VMs located on the ESXi host that you will use for scanning malware during the clean restore. Provide the *User name* and *Password* for each VM you select. 
. Select **Add** to create the isolated recovery environment. Ransomware Resilience tests the connection after you select add; This process can take a while.

== Perform a clean restore

. In the **Recovery** section, select the workload you want to restore. Select **Restore**. 
. In the Restore Type options, select **Clean restore** then **Next**. 
. Select the isolated recovery environment you want to use then **Next**. 
. Review the isolated recovery environment, confirming it's the correct location. Select **Restore**. 
. Ransomware Resilience performs the necessary set up. Once the set up completes successfully, Ransomware Resilience denotes **Setup complete**. To proceed to analysis, select **Next**. 
. After the setup completes, select **Run analysis**. Ransomware Resilience analyzes all available restore points in the seven days prior to the ransomware event, displaying the date and type of restore point. 
+
image:screenshot-clean-room-analysis.png[Screenshot of the clean restore analysis step.]
. After the analysis completes, select **Next** to plan your recovery. Ransomware Resilience presents two options: **Least data loss** and **Latest unimpacted restore point**. Select either option, and optionally look at individual file data for either restore point. 
+
To view granular file event data, select a file to analyze its encryption status; when it was created, modified, or deleted, and what restore point corresponds to what action. 
+
image:clean-restore-plan.png[Screenshot of the clean restore plan step.]
. After you choose your restore point, select **Next** to begin cleaning your files. 
. Ransomware Resilience begins cleaning the workload. 
+
When the cleaning completes, select **Start recovery** to initiate the recovery. 
. Choose if you want to save the original workload. To forgo saving the original workload, select **No, replace the original workload**. To save it, select **Yes, save the original workload** then enter a new name for the workload.
. Select **Start recovery** to initiate the recovery. 
. When the recovery completes, select **Next** to move to the final phase. 
. Select **Re
. Select **Release resources and end** to release the resources and close the clean room. To confirm you want to release the resource, select **End**. 

== Delete an isolated recovery environment

You can't delete a clean room with an restoration in progress. You can cancel the restore operation or wait until the restore completes then delete the clean room. 

. Go to **Settings**.
. In the Clean restore card, select **Manage**. 
. Identify the clean room you want to delete. In the actions menu column for that room, select the three dots `...` then **Delete**.
. In the dialog, select **Delete** to confirm the operation.
